# TCP 三次握手详解

TCP 三次握手是建立可靠连接的关键步骤。本文将用图解的方式详细讲解三次握手的全过程。

## 什么是三次握手？

三次握手（Three-Way Handshake）是 TCP 协议建立连接的过程，确保客户端和服务器都能正常收发数据。

## 三次握手过程

### 第一次握手（SYN）

客户端向服务器发送 SYN 包，表示请求建立连接。

### 第二次握手（SYN-ACK）

服务器收到 SYN 后，发送 SYN-ACK 包，表示同意建立连接。

### 第三次握手（ACK）

客户端收到 SYN-ACK 后，发送 ACK 包，确认连接建立。

## 为什么需要三次握手？

三次握手可以防止已失效的连接请求报文段突然传送到服务器，从而产生错误。

## 总结

三次握手确保了 TCP 连接的可靠性，是网络通信的基础。
